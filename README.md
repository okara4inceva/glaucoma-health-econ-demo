# Glaucoma Health Economics Demo

Rapid health economics demo evaluating early glaucoma detection in an ophthalmology setting.

This project compares usual care with a structured early detection pathway using simulated patient-level data. It estimates additional early glaucoma cases detected, advanced cases avoided, direct budget impact, productivity loss avoided, and sensitivity analysis.

## Important note

This is a demo project. The dataset is fully simulated and does not contain real patient data.

## Project package

The full GitHub-ready project package is uploaded as a ZIP file in this repository. It includes:

- simulated glaucoma patient dataset
- Python scripts
- base-case budget impact model
- cost-consequence analysis
- one-way sensitivity analysis
- output tables and charts
- CRF template
- real-world data collection plan

## Health economics angle

The project demonstrates how early glaucoma detection can be evaluated from a health economics perspective by estimating:

- additional early glaucoma cases detected
- advanced cases avoided
- direct budget impact
- productivity loss avoided
- cost per additional early case detected

## Real-world data plan

In a real hospital research setting, assumptions would be replaced with de-identified data from:

- EHR
- ophthalmology clinic records
- OCT systems
- visual field systems
- pharmacy data
- billing / finance data
- patient-reported outcomes

Real-world use would require IRB approval, data governance clearance, de-identification, and clinician validation.
